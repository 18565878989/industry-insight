# 产业链洞察报告系统 - 新功能开发

## 需求清单

### 1. 系统配置模块
- 模型API配置（API Key、Endpoint）
- 知识图谱配置（数据源、超参数）
- 本体配置（ontology设置）

### 2. 知识图谱数据导入
- CSV/JSON 批量导入
- API 对接（外部数据源）
- 去重和合并逻辑

### 3. 本体数据导入
- Ontology文件导入（OWL/RDF/JSON）
- 概念层次结构
- 关系类型定义

## 技术方案

```
┌─────────────────────────────────────────────────────────────┐
│                      系统配置模块                             │
├─────────────────────────────────────────────────────────────┤
│  配置表: model_configs, knowledge_graph_configs             │
│  API: GET/PUT /api/configs                                │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      数据导入模块                             │
├─────────────────────────────────────────────────────────────┤
│  知识图谱导入: POST /api/imports/knowledge-graph           │
│  API对接: POST /api/imports/from-api                      │
│  本体导入: POST /api/imports/ontology                      │
└─────────────────────────────────────────────────────────────┘
```

## 实现步骤

1. 创建配置模型 (model_config, kg_config, ontology_config)
2. 创建配置API路由
3. 创建数据导入服务
4. 创建导入API路由
5. 前端配置页面
