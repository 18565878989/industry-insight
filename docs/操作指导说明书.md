# 产业链洞察系统 - 操作指导说明书

## 📋 概述

产业链洞察是一个基于 SemiKong 本体知识库的半导体行业智能分析平台。

### 核心功能

- **SemiKong 本体知识库** - 712个半导体产业链概念
- **AI 智能问答** - 基于 MiniMax-M2.7 大模型
- **公司数据管理** - 100+ 半导体公司信息
- **知识图谱可视化** - 交互式产业链关系图
- **供应链分析** - 产业链价值链视图
- **趋势分析** - 市场趋势与洞察

---

## 🚀 快速开始

### 1. 启动服务

```bash
# 启动后端 (端口 5010)
cd ~/.openclaw/workspace/industry-insight/backend
source venv/bin/activate
python -c "from app import create_app; app = create_app(); app.run(host='127.0.0.1', port=5010, debug=False, use_reloader=False)"

# 启动前端 (端口 3000)
cd ~/.openclaw/workspace/industry-insight/frontend
npm run dev
```

### 2. 访问应用

- **前端**: http://localhost:3000
- **后端 API**: http://localhost:5010/api

## 📱 页面功能

### Dashboard (首页)
- 产业概览统计
- 快速入口链接
- 最近公司列表
- 行业分类摘要

### Companies (公司列表)
- 浏览所有公司
- 搜索公司名称
- 按行业分类筛选
- 查看公司详情

### Company Detail (公司详情)
- 公司基本信息
- 产品与竞争对手
- 供应链关系
- 在知识图谱中的位置

### Knowledge Graph (知识图谱)
- D3.js 交互式可视化
- 按分类筛选节点
- 点击查看详情
- 缩放与拖拽

### Supply Chain (供应链视图)
- 产业链阶段展示
- 各环节代表公司
- 关系可视化

### Analysis (趋势分析)
- 市场趋势图表
- 行业数据洞察
- 竞争分析

### AI Assistant (智能问答) 🤖
- 半导体专业知识问答
- 基于 SemiKong 本体
- 建议问题快捷按钮

**访问**: http://localhost:3000/assistant

### Settings (系统配置)
- **Model Config**: LLM 模型配置
- **Ontology**: SemiKong 本体状态
- **Data Sources**: 数据源管理

## 🔧 配置说明

### MiniMax API Key 配置

2. 输入 API Key
3. 保存配置

**当前配置**:
- Provider: minimax
- Model: MiniMax-M2.7
- Endpoint: https://api.minimaxi.com/v1

## 🔌 API 接口

### 健康检查
```
GET http://localhost:5010/api/health
```

### 公司列表
```
GET http://localhost:5010/api/companies
```

### 公司详情
```
GET http://localhost:5010/api/companies/{id}
```

### 知识图谱
```
GET http://localhost:5010/api/knowledge-graph
```

### AI 问答
```
POST http://localhost:5010/api/llm/chat
Content-Type: application/json

{
  "question": "什么是光刻？",
  "use_ontology": true
}
```

### LLM 状态
```
GET http://localhost:5010/api/llm/status
```

## 🛠 故障排查

### API 返回 403 Forbidden
- 检查 CORS 配置
- 确认前端端口与后端匹配

### AI Assistant 无响应
- 检查 MiniMax API Key 是否配置
- 确认 API Key 有效（格式：sk-cp-xxx）

### 前端页面空白
- 检查 Node.js 版本
- 运行 `npm install` 重新安装依赖
- 运行 `npm run build` 重新构建

### 后端无法启动
- 检查端口占用
- 确认虚拟环境激活
- 查看错误日志
